# Test Files — synthetic EDI library

Synthetic X12 4010 files for manual drop-folder testing. Generated by `node scripts/generate-test-edi.mjs`.

## Trading partner IDs

| Role | ISA ID |
|------|--------|
| Vendor (sender) | `VENDOR01` |
| Your hub (receiver) | `EDIHUB` |

If your wizard partner uses different ISA IDs (e.g. `ACME`), either update **Partners** config or regenerate after we align IDs to your real samples.

## Quick start

1. Drop `lifecycles/PO-10001/01_850_purchase_order.edi` into `C:\EDI\Inbound`
2. Confirm it appears under **Transactions** and moves to `C:\EDI\drop-processed`
3. Drop the rest of the PO-10001 folder in order (02 → 05)
4. Search **Lifecycle** or **Search** for `PO-10001`

## Folder layout

| Folder | Purpose |
|--------|---------|
| `lifecycles/` | Related documents keyed by PO number — drop in numeric order |
| `singles/` | One-off files for quick smoke tests |
| `edge-cases/` | Batched interchanges, line wrapping, dedup behavior |
